# 기획자 문서

## 1. 프로젝트 개요

이 프로젝트는 현재 재산, 정기적인 수입/지출, 카드 사용내역을 수집하고 분석해서 일별 현금흐름을 시각적으로 보여주는 시스템을 만드는 것을 목표로 한다.

핵심 방향:

- `n8n`을 데이터 수집, 정규화, 집계 자동화의 중심으로 사용
- `Postgres`를 금융 데이터 저장소로 사용
- 외부 네트워크에서 휴대폰으로 접속 가능한 웹 환경 제공
- 초기 단계에서는 운영 가능한 기반 세팅과 데이터 모델을 먼저 구축

## 7. 사용자 시나리오

이 시스템의 1차 사용자 행동 흐름은 아래 순서로 정의한다.

1. 자산 계좌와 카드 정보를 등록한다.
2. 카드 사용내역 CSV 또는 수동 입력으로 거래를 업로드한다.
3. 시스템이 거래를 정규화하고 분류한다.
4. 사용자가 분류 결과를 검수하고 필요한 항목을 수정한다.
5. 일별 현금흐름, 잔액 변화, 카테고리별 지출을 조회한다.

## 8. 입력 데이터 명세 초안

1차 MVP에서 우선 지원할 입력 방식은 카드 사용내역 CSV 업로드다.

최소 필요 컬럼:

- 거래일시
- 사용금액
- 가맹점명
- 승인번호 또는 고유 거래 식별자
- 결제수단 또는 카드명

추가 가능 컬럼:

- 할부 여부
- 취소 여부
- 메모
- 카드사 분류 값

입력 정책:

- 동일 소스에서 동일 거래 식별자가 있으면 중복 후보로 처리한다.
- 필수 컬럼 누락 시 적재하지 않고 오류로 분리한다.
- CSV 포맷은 카드사별로 다를 수 있으므로 파서 레이어를 분리한다.

## 9. 거래 분류 기준 초안

거래는 최소 아래 3개 유형으로 분류한다.

- `income`: 급여, 입금, 환급 등 순유입
- `expense`: 소비, 수수료, 자동이체 등 순유출
- `transfer`: 본인 계좌 간 이동, 카드대금 이체 등 내부 이동

분류 원칙:

- 실제 자산 총액이 변하지 않는 이동은 `transfer`로 본다.
- 외부에서 들어온 금액은 `income`으로 본다.
- 외부로 나간 지출은 `expense`로 본다.
- 자동 분류가 애매한 경우 검수 대상 큐로 보낸다.

## 10. 대시보드 지표 정의 초안

1차 대시보드에서 보여줄 핵심 지표:

- 오늘 지출 합계
- 오늘 순현금흐름
- 이번 달 누적 지출
- 카테고리별 지출 합계
- 계좌별 최신 잔액
- 최근 거래 내역

기준 원칙:

- 일별 현금흐름은 `occurred_on` 기준으로 계산한다.
- 순현금흐름은 `income - expense`로 계산한다.
- `transfer`는 자산 이동 분석에는 포함하되 순현금흐름 합계에서는 기본 제외한다.
- 지표 계산식은 실제 워크플로우 구현 전에 문서로 확정한다.

## 11. 운영 및 예외 정책 초안

- 중복 거래는 자동 병합하지 않고 우선 중복 후보로 표시한다.
- 재집계는 일별 배치 실행 시 최근 N일을 다시 계산하는 방식으로 설계한다.
- 누락 데이터가 확인되면 원본 적재 후 정규화와 일별 집계를 다시 실행한다.
- 분류 실패 건수와 적재 실패 건수는 운영자가 확인할 수 있어야 한다.
- 수동 수정 이력은 추후 별도 테이블 또는 감사 로그로 확장한다.
