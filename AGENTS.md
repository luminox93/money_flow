# 에이전트 작업 지시사항

## 커밋 규칙
- 커밋은 반드시 기능 단위로 나눈다.
- 서로 다른 목적의 변경(예: 기능 추가 + 리팩터링 + 포맷 수정)은 한 커밋에 섞지 않는다.
- 커밋 메시지는 반드시 Angular(Conventional Commits) 형식을 따른다.
- 커밋 메시지는 반드시 한국어로 작성한다.

## 커밋 메시지 형식
`<type>(<scope>): <한국어 요약>`

### type 예시
- `feat`: 기능 추가
- `fix`: 버그 수정
- `refactor`: 동작 변화 없는 구조 개선
- `docs`: 문서 변경
- `test`: 테스트 추가/수정
- `chore`: 빌드/설정/기타 작업
- `style`: 코드 포맷팅(동작 변경 없음)
- `perf`: 성능 개선
- `ci`: CI 설정 변경
- `build`: 빌드 시스템/의존성 변경
- `revert`: 이전 커밋 되돌리기

## 커밋 메시지 예시
- `feat(payment): 자동이체 일정 등록 기능 추가`
- `fix(auth): 토큰 만료 시 재로그인 루프 오류 수정`
- `refactor(flow): 송금 상태 계산 로직 분리`
- `chore(git): 에이전트 관련 파일 gitignore 규칙 추가`
